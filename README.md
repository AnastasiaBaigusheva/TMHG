# Самая сложная игра в мире — MVP

Лендинг + чат с ИИ для психологической игры-исследования. Сделано на Next.js 14 (App Router), TypeScript, Tailwind CSS, использует Anthropic Claude Messages API со стримингом.

**Это не медицинский продукт, не терапия, не диагностика и не замена психолога.**

## 1. Установка зависимостей

```bash
npm install
```

## 2. Переменные окружения

Скопируйте файл `.env.example` в `.env.local`:

```bash
cp .env.example .env.local
```

Заполните переменные:

```
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-sonnet-4-6
NEXT_PUBLIC_FEEDBACK_URL=
```

- `ANTHROPIC_API_KEY` — обязателен, ключ от console.anthropic.com. Никогда не публикуется на клиенте, используется только в `app/api/chat/route.ts` на сервере.
- `ANTHROPIC_MODEL` — название модели Claude. Если не задано, используется значение по умолчанию из `lib/anthropic.ts` (`DEFAULT_MODEL`).
- `NEXT_PUBLIC_FEEDBACK_URL` — ссылка на форму обратной связи (например, Google Forms или Tally). Если оставить пустым, кнопка "Оставить отзыв" не будет показана после завершения игры.

## 3. Запуск локально

```bash
npm run dev
```

Откройте http://localhost:3000

## 4. Деплой на Vercel

1. Запушьте проект в GitHub репозиторий.
2. Зайдите на vercel.com → New Project → импортируйте репозиторий.
3. В настройках проекта (Settings → Environment Variables) добавьте:
   - `ANTHROPIC_API_KEY`
   - `ANTHROPIC_MODEL`
   - `NEXT_PUBLIC_FEEDBACK_URL`
4. Нажмите Deploy.

Можно также использовать Vercel CLI:

```bash
npm i -g vercel
vercel
```

## 5. Где поменять системный промпт

Системный промпт находится в `lib/anthropic.ts`, константа `SYSTEM_PROMPT`. Там же:
- `DEFAULT_MODEL` — модель по умолчанию, если `ANTHROPIC_MODEL` не задана в env.
- `MAX_USER_MESSAGE_LENGTH` — лимит длины сообщения пользователя (по умолчанию 4000 символов).
- `CRISIS_KEYWORDS` — список ключевых слов для базовой клиентской проверки на кризисные сообщения (дополнительно к логике в системном промпте).

## 6. Где поменять тексты лендинга

Все тексты лендинга находятся в `components/Landing.tsx`: заголовок, подзаголовок, описание, блок "Как это работает", блок "Важно" и CTA-кнопки. Стили оформлены через Tailwind-классы прямо в JSX.

Карта зон исследования (8 зон: Запрос, Ценности, Страхи, Защиты, Отношения, Сценарии, Ресурсы, Инсайты) находится в `components/Map.tsx`, константа `ZONES`.

## Структура проекта

```
app/
  page.tsx              — главная страница (рендерит Landing)
  chat/page.tsx          — страница чата (рендерит Chat)
  api/chat/route.ts      — серверный route, вызывает Anthropic API со стримингом
  layout.tsx, globals.css
components/
  Landing.tsx            — лендинг
  Chat.tsx               — интерфейс чата, логика стрима, пауза/завершение, защита от пустых/слишком длинных сообщений, базовое определение кризисных сообщений
  Map.tsx                — визуальная карта из 8 зон
lib/
  anthropic.ts           — системный промпт, константы, проверка кризисных формулировок
```

## Важные ограничения MVP

- История чата хранится только в состоянии браузера (в памяти React) — при перезагрузке страницы чат начинается заново. Базы данных нет.
- API-ключ используется только на сервере (`app/api/chat/route.ts`), никогда не передается на клиент.
- При обнаружении в сообщении пользователя слов, связанных с самоповреждением/суицидом (см. `CRISIS_KEYWORDS` в `lib/anthropic.ts`), интерфейс показывает предупреждение и ставит игру на паузу. Основная логика реагирования на кризис заложена в системном промпте модели.
- Подсветка зон карты на фронтенде происходит по простому совпадению ключевых слов в ответе ассистента (название зоны на русском в тексте) — это упрощенная эвристика для MVP, не точный парсинг.

## Что можно добавить дальше (вне MVP)

- Сохранение истории в базе данных и аутентификация пользователей.
- Более надежное определение текущей зоны карты (например, через структурированный JSON-ответ модели вместо парсинга текста).
- Полноценная модерация сообщений пользователя.
- Аналитика и метрики прохождения игры.
